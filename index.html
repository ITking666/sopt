Introduction

SOPT is an open-source code that provides functionality to perform sparse optimisation using state-of-the-art convex optimisation algorithms. Furthermore, SOPT implements the SARA algorithm, where the average sparsity of images over a set of dictionaries is exploited.

SOPT is written in C primarily but also contains partial Matlab support. SOPT is largely provided to support the PURIFY package, a companion open-source code to perform radio interferometric imaging, also written by the authors of SOPT. For further background please see the following paper.

This documentation outlines how to obtain SOPT, before describing installation details, source code documentation and Matlab support. Reference, license and author information then follows.

Download

SOPT is an open-source package available under the license described below.

The SOPT code is hosted by github and available here.

Installation

The SOPT code is written largely in C, hence a suitable C compiler will be required to build the main code. In the following the dependencies of SOPT are described, before outlining instructions to compile, test, self-document and clean the code.

Dependencies

The SOPT library itself requires only the LibTIFF package. However, the FFTW package is required to build the SOPT example programs. Please ensure the appropriate dependencies are installed on your system before attempting to build SOPT.

Compiling

A makefile is provided to build the code. Before compiling you will need to edit the makefile to specify your compiler and the location where the dependencies are installed (if all dependencies are installed in the same parent directory as SOPT then it may not be necessary to edit the makefile).

Once you have set the makefile up for your system, SOPT may be compiled by running: 

 >> make all

This will build the library, test program, and demonstration programs. If your system is configured to build all of these components then you are done. If you prefer to build only a subset of these components then read on.

You may alternatively build the individual components of SOPT. To build the library only run: 

 >> make lib

To build the test program run: 

 >> make test

To build the demonstration programs run: 

 >> make demos

Testing

Run the test program to verify your installation. A default test may be performed by running: 

 >> make runtest

If SOPT is installed correctly the test program should run and print test results to the terminal.

Alternatively, you may run the test program directly by running: 

 >> ./bin/sopt_test

To check the version number of your installation run: 

 >> ./bin/sopt_about

Documentation

The SOPT code is self-documenting. Although documentation should be available in a standard checkout of the code, html source documentation can be generated by running: 

 >> make doc

Documentation is generated using doxygen, thus you must have doxygen installed on your system to generate the source documentation.

Cleaning up

To clean up your version of the SOPT code and remove all builds run: 

 >> make clean

To remove all source documentation run: 

 >> make cleandoc

Matlab

A separate Matlab implementation is provided with SOPT. This implementation includes some (but not all) of the optimisation algorithms implemented in the C code, including the SARA algorithm.

The Matlab implementation is contained in the matlab directory. This is a stand-alone implementation and does not call any of the C code. In future, Matlab interfaces to the C code may also be included in SOPT.

See matlab/README.txt for an overview of the Matlab implementation. The stand-alone Matlab implementation is also self-documenting; corresponding documentation can be found in matlab/doc. We thank Gilles Puy for contributing to this Matlab implementation.
